# 大模型评测 Prompts

## 使用说明

1. 复制下方的 Prompt 1 或 Prompt 2
2. 投喂给待评测的大模型
3. 等待模型完整回答
4. 保存回答并评分

---

## ⭐ 重要说明：评测基准模型

**doubao-seed-code-dogfood-2.1.1** 是评测用的**原型基准模型**。

### 评测规则

- ✅ **所有评测的其他模型需要与 doubao-seed-code-dogfood-2.1.1 比对**
- ✅ **分析各自优劣和产出报告**
- ❌ **其他所有模型之间不需要比对和产出报告**

### 评测流程

1. 首先评测 doubao-seed-code-dogfood-2.1.1，建立基准
2. 评测其他模型时，与 doubao-seed-code-dogfood-2.1.1 进行对比
3. 分析相对于基准模型的优势和劣势
4. 产出对比报告

### 示例

- ✅ 正确：GLM-5 vs doubao-seed-code-dogfood-2.1.1
- ✅ 正确：Kimi vs doubao-seed-code-dogfood-2.1.1
- ❌ 错误：GLM-5 vs Kimi（不需要）

## Prompt 1：基础浏览器自动化操作

```
这是一个基于 Playwright MCP 的浏览器自动化任务场景，评测大模型执行基础浏览器操作的能力。

任务目标

使用 Playwright MCP 工具完成以下基础操作：

1. 页面导航
   - 访问示例页面 https://demo.playwright.dev/todomvc
   - 验证页面标题正确
   - 等待页面完全加载

2. 元素交互
   - 点击"Add"按钮添加待办事项
   - 输入文本："Buy groceries"
   - 按 Enter 键提交
   - 验证待办事项已添加

3. 表单操作
   - 填写登录表单（用户名：testuser，密码：password123）
   - 点击登录按钮
   - 验证登录成功

4. 动态内容处理
   - 点击按钮触发异步加载
   - 等待新内容出现（使用 browser_wait_for）
   - 验证加载后的内容

5. 数据提取
   - 执行 JavaScript 获取页面数据
   - 提取页面中所有链接的文本和 URL
   - 以 JSON 格式输出

技术要求

- 使用语义化定位器（getByRole、getByText、getByLabel 等）
- 添加适当的等待逻辑（避免固定延迟）
- 生成的代码需可直接运行
- 包含基本的错误处理

输出要求

1. 完整的 Playwright 测试代码
2. 每个步骤的执行结果
3. 最终提取的数据（JSON 格式）
```

## Prompt 2：电商购物流程自动化

```
这是一个基于 Playwright MCP 的电商购物流程自动化任务，评测大模型完成复杂业务流程的能力。

任务目标

实现一个完整的电商购物流程自动化脚本：

1. 商品浏览
   - 访问电商演示站点
   - 浏览商品分类（Electronics、Clothing 等）
   - 进入商品详情页
   - 提取商品信息（名称、价格、评分）

2. 购物车管理
   - 将商品添加到购物车
   - 查看购物车
   - 修改商品数量
   - 验证总金额计算正确

3. 结算流程
   - 进入结算页面
   - 填写收货地址表单
   - 选择配送方式
   - 应用优惠码（如有）
   - 提交订单

4. 订单验证
   - 验证订单确认页面
   - 提取订单号
   - 查看订单详情

5. 订单管理
   - 查看订单历史
   - 找到最新订单
   - 验证订单状态

技术要求

- 使用 MCP 工具：browser_navigate、browser_click、browser_type、browser_evaluate、browser_snapshot 等
- 正确处理动态加载内容
- 处理可能出现的弹窗和对话框
- 生成的代码需符合 Playwright 最佳实践

输出要求

1. 完整的购物流程代码
2. 每个关键步骤的截图或快照
3. 提取的订单信息（JSON 格式）
4. 执行过程中遇到的问题及解决方案
```
