# Playwright MCP 大模型评测框架

## 一、评测概述

**评测目标**：评估各大模型在浏览器自动化场景下的综合能力  
**评测工具**：Playwright MCP 服务器  
**测试环境**：Playwright 1.59.0-next

---

## 二、评测维度与评分标准

### 1. 用户体验满意度（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 超出预期，完全无需人工干预 |
| 4 | 满足需求，仅需少量调整 |
| 3 | 基本可用，需要中等程度修改 |
| 2 | 问题较多，需要大量修改 |
| 1 | 无法使用，需要重新实现 |

**评分项**：
- 代码可执行性
- 回答清晰度
- 问题解决完整性
- 交互友好性

---

### 2. 规划执行反馈（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 清晰的步骤规划，每步都有明确反馈 |
| 4 | 有步骤规划，大部分有反馈 |
| 3 | 有基本规划，反馈不完整 |
| 2 | 规划模糊，反馈很少 |
| 1 | 无规划，无反馈 |

**检查点**：
- [ ] 是否先列出实现步骤
- [ ] 每步执行后是否有状态说明
- [ ] 遇到问题是否有错误分析
- [ ] 完成后是否有总结

---

### 3. 理解推理能力

#### 3.1 代码理解（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 代码完全正确，符合最佳实践 |
| 4 | 代码基本正确，有小问题 |
| 3 | 代码逻辑正确，有语法错误 |
| 2 | 代码逻辑有部分错误 |
| 1 | 代码完全不可用 |

**检查点**：
- [ ] Playwright API 使用正确
- [ ] 定位器选择合理
- [ ] 等待逻辑正确
- [ ] 错误处理完善
- [ ] 代码结构清晰

#### 3.2 自然语言理解（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 完全理解所有需求，无遗漏 |
| 4 | 理解大部分需求，遗漏次要点 |
| 3 | 理解基本需求，有遗漏 |
| 2 | 理解有偏差，遗漏较多 |
| 1 | 完全误解需求 |

**检查点**：
- [ ] 识别所有功能要求
- [ ] 理解约束条件
- [ ] 正确解读隐含需求

---

### 4. 复杂指令遵循（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 遵循所有指令，包括细节要求 |
| 4 | 遵循主要指令，忽略次要细节 |
| 3 | 遵循基本指令，遗漏部分要求 |
| 2 | 只遵循部分指令 |
| 1 | 基本不遵循指令 |

**检查点**：
- [ ] 输出格式符合要求
- [ ] 技术栈选择正确
- [ ] 性能要求被满足
- [ ] 注意事项被遵守

---

### 5. 工程完备度（5 分制）

| 分数 | 标准 |
|------|------|
| 5 | 生产级代码，文档齐全，测试完整 |
| 4 | 代码完整，有基本注释 |
| 3 | 代码基本完整，缺少注释 |
| 2 | 代码不完整，缺少关键部分 |
| 1 | 代码碎片，无法运行 |

**检查点**：
- [ ] 代码完整性
- [ ] 注释充分性
- [ ] 错误处理
- [ ] 边界条件处理
- [ ] 可维护性

## 三、问题类型分类

### 问题记录表

| 问题类型 | 定义 | 示例 | 记录 |
|----------|------|------|------|
| **幻觉** | 编造不存在的 API 或功能 | 使用不存在的 Playwright 方法 | ⬜ |
| **上下文丢失** | 忘记之前的对话内容 | 重复询问已提供的信息 | ⬜ |
| **指令遵循失效** | 未按要求执行 | 忽略输出格式要求 | ⬜ |
| **死循环** | 重复相同的操作或回答 | 反复尝试相同的错误代码 | ⬜ |
| **偷懒** | 省略关键步骤或代码 | 使用 `// ... rest of code` 省略 | ⬜ |
| **代码破坏** | 修改了不应修改的部分 | 删除了原有的关键功能 | ⬜ |
| **代码 Bug** | 代码有错误无法运行 | 语法错误、逻辑错误 | ⬜ |
| **废话过多** | 包含大量无用内容 | 重复解释、过度道歉 | ⬜ |
| **输出中断** | 回答不完整被截断 | 代码写到一半结束 | ⬜ |
| **目标漂移** | 偏离原始任务目标 | 开始解决其他问题 | ⬜ |
| **分心** | 注意力转移到次要问题 | 过度优化非核心功能 | ⬜ |
| **其他** | 未归类的问题 | - | ⬜ |

## 五、评测流程

### 1. 准备阶段

```bash
# 安装 Playwright
npm install -D @playwright/test@next

# 安装浏览器
npx playwright install

# 准备测试环境
- 确保网络畅通
- 准备测试账号
- 清空浏览器缓存
```

### 2. 测试执行

对于每个模型：

1. **投喂 Prompt**
   - 完整发送 Prompt 1
   - 等待模型完整回答
   
2. **代码验证**
   - 复制生成的代码
   - 创建测试文件
   - 运行测试
   - 记录执行结果

3. **评分记录**
   - 逐项评分
   - 记录问题
   - 截图保存关键输出

4. **重复测试**
   - 对 Prompt 2 重复上述步骤

### 3. 结果分析

- 计算每个模型的平均分
- 统计问题类型分布
- 识别常见问题模式
- 生成对比报告

### B. 评分示例

**5 分代码示例**：
```typescript
import { test, expect } from '@playwright/test';

test('complete shopping flow', async ({ page }) => {
  // 步骤 1: 导航到主页
  await page.goto('https://demo.playwright.dev');
  await expect(page).toHaveTitle(/Demo/);
  
  // 步骤 2: 登录
  await page.getByRole('link', { name: 'Sign in' }).click();
  await page.getByLabel('Username').fill('demo');
  await page.getByLabel('Password').fill('demo123');
  await page.getByRole('button', { name: 'Sign in' }).click();
  
  // 验证登录成功
  await expect(page.getByText('Welcome')).toBeVisible();
});
```

**1 分代码示例**：
